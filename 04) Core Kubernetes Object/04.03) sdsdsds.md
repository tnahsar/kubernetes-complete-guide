
# 📘 First: Important Definitions

## 1️⃣ Linux Namespaces

These are:

```text id="x1m’wini"
Linux kernel features
```

Examples:

* network namespace
* PID namespace
* UTS namespace (hostname)
* IPC namespace
* mount namespace

They provide isolation.

---

# 6️⃣ Container Runtime Creates Linux Namespaces

The runtime asks Linux kernel to create namespaces for the Pod.

Examples:

| Linux Namespace   | Purpose            |
| ----------------- | ------------------ |
| Network namespace | Pod networking     |
| UTS namespace     | Hostname isolation |
| IPC namespace     | IPC isolation      |
| PID namespace     | Process isolation  |


| Namespace Type                      | Purpose                              |
| ---------------------------         | ------------------------------------ |
| Network namespace                   | Isolates Pod networking              |
| PID namespace                       | Isolates process IDs                 |
| Mount namespace                     | Isolates filesystem mounts           |
| IPC namespace                       | Isolates inter-process communication |
| UTS namespace (hostname namespace)  | Isolates hostname/domain name        |
| User namespace *(optional)*         | Isolates user/group IDs              |

These namespaces collectively become part of:

```text id="k5m’wini"
the Pod sandbox environment
```

---

# 7️⃣ Container Runtime Starts Pause Container

Now runtime creates:

```text id="d3m’wini"
pause container
```

The pause container:

* joins these namespaces
* stays running
* becomes the stable process attached to them

This is VERY important because Linux namespaces must belong to a running process.

So:

```text id="y7m’wini"
pause container keeps the Pod namespaces alive
```

especially:

* network namespace
* UTS namespace
* IPC namespace

---

# 8️⃣ CNI Plugin Configures Pod Networking

Now:

```text id="p1m’wini"
CNI plugin
```

Examples:

* Calico
* Flannel
* Cilium

does:

* create virtual interfaces
* assign Pod IP
* configure routing

The Pod IP belongs to:

```text id="c6m’wini"
the Pod network namespace
```

which the pause container is already holding alive.

---

# 9️⃣ Application Containers Start

Now runtime starts actual containers:

```text id="t4m’wini"
nginx container
sidecar container
```

These containers:

```text id="x8m’wini"
join the SAME Pod namespaces
```

already held by pause container.

So they share:

* same Pod IP
* same localhost
* same hostname
* same network namespace

---

# 🔟 Pod Becomes Running

Now:

```text id="f2m’wini"
Pod status = Running
```

# 🌐 Why Pod Sandbox Is Important

Without sandbox:

* containers would not share networking
* Pod abstraction would not work
* localhost communication would fail

Sandbox is what makes:

```text
multiple containers behave like one logical Pod
```
---




All containers inside a Pod must share:

* network namespace
* Pod IP
* localhost
* IPC namespace (sometimes)

## 🌐 Pod Networking

Once the pause container is created and the pod network namespace is established, all application containers inside the Pod join that same network namespace.

### Example:

Suppose Pod contains:

```text
nginx container
log-agent container
```

```text
Pause container
      ↓
Owns Pod's shared namespace
      ↓
nginx container joins it
log-agent container joins it
```

Now all containers share:

* same Pod IP
* same localhost
* same port space

# 📘 First Understand What a Network Namespace Is

A network namespace contains:

* Pod IP
* network interfaces
* routing table
* localhost
* port space

Example:

```text
Pod IP = 10.244.0.5
```

and:

```text 
localhost
```

and:

```text
port 8080
```

all belong to the Pod’s network namespace.

---

# 🌐 Real Meaning of “Port Space”

When we say:

```text id="x6m’wini"
shared port space
```

we mean:

all containers inside same Pod share:

* same TCP/UDP ports.

Example:

If nginx container uses:

```text id="q1m’wini"
port 8080
```

another container inside same Pod CANNOT also use:

```text id="f8m’wini"
port 8080
```

because:

* they share same network namespace.
