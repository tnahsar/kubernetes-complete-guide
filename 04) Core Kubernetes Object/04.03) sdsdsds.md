# 🌐 Why Pod Sandbox Is Important

Without sandbox:

* containers would not share networking
* Pod abstraction would not work
* localhost communication would fail

Sandbox is what makes:

```text
multiple containers behave like one logical Pod
```

---

## 🌐 Pod Networking

Once the pause container is created and the pod network namespace is established, all application containers inside the Pod join that same network namespace.

### Example:

Suppose Pod contains:

```text
nginx container
log-agent container
```

```text
Pause container
      ↓
Owns Pod's shared namespace
      ↓
nginx container joins it
log-agent container joins it
```

Now all containers share:

* same Pod IP
* same localhost
* same port space

---

# 📘 First Understand What a Network Namespace Is

A network namespace contains:

* Pod IP
* network interfaces
* routing table
* localhost
* port space

Example:

```text
Pod IP = 10.244.0.5
```

and:

```text 
localhost
```

and:

```text
port 8080
```

all belong to the Pod’s network namespace.

---

# 🌐 Real Meaning of “Port Space”

When we say:

```text id="x6m’wini"
shared port space
```

we mean:

all containers inside same Pod share:

* same TCP/UDP ports.

Example:

If nginx container uses:

```text id="q1m’wini"
port 8080
```

another container inside same Pod CANNOT also use:

```text id="f8m’wini"
port 8080
```

because:

* they share same network namespace.
